# ブラウザゲーム仕様書

## 1. 概要

本作は、ブラウザ上で動作する2D横スクロールのランアクションゲーム。

プレイヤーキャラクターは自動で前進し、ユーザーはジャンプ、ダッシュ、攻撃を使って障害物や敵を避けながら進行する。

## 2. 開発環境

* 言語: TypeScript
* ビルドツール: Vite
* ゲームエンジン: Phaser 3
* デプロイ: Cloudflare Workers / Pages
* デプロイツール: Wrangler

## 3. 権利方針

既存ゲームの画像・音声・キャラクター・ロゴ等の素材は使用しない。

参考にするのは以下に限定する。

* 高速な横スクロールアクションの操作感
* ジャンプ、ダッシュ、攻撃の組み合わせ
* 2Dドット絵風の表現
* ステージ攻略型のゲームフィール

素材、キャラクター、世界観、名称はすべてオリジナルとする。

## 4. 世界観・デザイン方針

### 4.1 世界観

* 世界観はメカニカルなSFテイストをベースとする
* ロボット、機械、エネルギー兵器などの要素を中心に構成する
* 工場、基地、機械都市などを連想させる背景表現を想定する
* 質感としては金属、発光、機械構造などを重視する
* 既存作品のキャラクターやデザインを直接流用しない範囲で、レトロアクションゲームらしい雰囲気を目指す

### 4.2 操作体験方針

* ブラウザゲームとして、過度に複雑な操作にはしない
* 少ないボタン数で直感的に遊べることを重視する
* 高速アクション感は維持しつつ、理不尽な難しさには寄せすぎない
* 地形への挟まりや即死など、強いストレス要素は初期段階では避ける
* プレイヤーの操作精度だけではなく、ビルド選択や状況判断も楽しさとして成立させる

## 5. ゲーム内容

### 5.1 基本コンセプト

本作は、長距離の到達を目指す2D横スクロール型のランアクションゲーム。

プレイヤーキャラクターは画面上の一定位置に表示され、キャラクター自身が画面内を大きく移動するのではなく、背景や敵が右から左へ流れることで前進を表現する。

ユーザーはジャンプ、ダッシュ、通常攻撃、チャージショットを使い、敵を倒す、または回避しながらできるだけ長く進み続ける。

### 5.2 基本ルール

* プレイヤーは放置状態でも自動で進行する
* 背景や敵がスクロールすることで移動を表現する
* プレイヤーはジャンプできる
* プレイヤーはダッシュできる
* プレイヤーは通常攻撃とチャージショットを使える
* 敵や敵弾に当たるとダメージを受ける
* プレイヤーのHPが0になるとゲームオーバー
* 基本的にはゴール到達ではなく、永遠に走り続けて長距離到達を目指す
* スコアは到達距離のみを基本とする
* 敵を倒すことで経験値リソースを獲得できる
* 通常敵は一定確率で回復アイテムをドロップする
* レベルアップによってプレイヤーを強化できる
* 一定距離ごとにボス戦が発生する

### 5.3 地形方針

初期仕様では、複雑なステージ地形や穴は扱わない。

理由は、強制スクロール中に地形に挟まって即死するような体験はストレスが大きいため。

初期は以下を中心にゲーム性を作る。

* 敵配置
* 敵の移動パターン
* 敵弾
* ジャンプによる回避
* ダッシュによる距離・攻撃効率の変化

## 6. 操作

| 操作   | キーボード     | 備考                                         |
| ---- | --------- | ------------------------------------------ |
| ジャンプ | Space / Z | 押している時間で高さが変化                              |
| ダッシュ | Shift / X | 一定時間スクロール速度が上がる                            |
| 攻撃   | C         | 押した瞬間に攻撃判定を行う。チャージ完了時はチャージショット、それ以外は通常弾を発射 |
| 一時停止 | Esc       | ポーズ画面を表示                                   |

## 7. プレイヤー仕様

### 7.1 状態

プレイヤー状態は単一の状態機械として扱うのではなく、複数の状態を同時に保持できる構成を前提とする。

例:

* Jump + Dash
* Jump + Charge
* Dash + Attack

などの組み合わせが同時に成立する。

プレイヤーは以下の状態を持つ。

* Run
* Jump
* Fall
* Dash
* Attack
* Charge
* ChargeShot
* Damage
* Dead

### 7.2 自動前進

* プレイヤーの画面上のX座標は基本的に固定する
* 通常時、背景・敵・弾・ステージ要素が右から左へ流れる
* 進行距離はスクロール量として管理する
* ダッシュ中はスクロール速度が上がる

### 7.3 ジャンプ

* 接地中のみジャンプ可能
* ジャンプボタンを押している時間によってジャンプ高度が変化する
* ジャンプ中にボタンを離すと上昇を弱める
* 短押しでは低く、長押しでは高く跳ぶ
* 二段ジャンプは初期仕様では扱わない

### 7.4 ダッシュ

* ダッシュ入力で一定時間スクロール速度が上がる
* ダッシュボタンを押している間、最大継続時間までダッシュする
* ダッシュは任意のタイミングでキャンセルできる
* ダッシュ中もジャンプ可能
* ダッシュ中にジャンプした場合、ダッシュの慣性を残したままジャンプする
* ダッシュ終了後、速度は通常速度へ戻る
* ダッシュにクールタイムは設けない
* ダッシュ終了後、次のダッシュを出せるまでに短い後隙を設ける
* ダッシュ後隙は時間経過のみで解除する
* ダッシュ後隙はジャンプの最短滞空時間より短い程度とする
* ダッシュ後隙が体感上見えるのは、主に地上で連続ダッシュを行う場合のみとする
* ダッシュジャンプを行った場合、ダッシュ後隙は発生しない
* ダッシュジャンプ後の着地から、すぐに再度ダッシュできる
* 初期実装では必須ではないが、ダッシュを含む主要操作は先行入力に対応する方針とする

### 7.5 通常攻撃

* 攻撃ボタンを押した瞬間に通常弾を発射する
* 通常弾は前方へ直進する
* 通常弾は画面内に最大3発まで存在できる
* 画面内の弾が3発に達している場合、新しい通常弾は発射できない
* 通常弾が敵に当たる、または画面外へ出ると消滅する
* 敵に接近して撃つほど、弾がすぐ消えるため実質的な連射効率が上がる
* 通常移動中より、ダッシュ中に撃った通常弾の方がやや攻撃力が高い
* 連打を前提にせず、押しっぱなし操作でも遊びやすい入力受付を目指す

### 7.6 チャージショット

* 攻撃ボタンを押していない間、チャージが進行する
* チャージ完了後に攻撃ボタンを押すとチャージショットを発射する
* 攻撃ボタンを押した瞬間のみが攻撃判定へ影響する
* 押しっぱなし状態と離しっぱなし状態そのものには意味を持たせない
* チャージショットは通常弾より高威力
* チャージ中も移動、ジャンプ、ダッシュは可能
* チャージショットのダメージに乱数は使用しない
* 何も操作していない時間にも攻撃準備が進むことで、操作負荷を下げる

### 7.7 HP・ダメージ

* プレイヤーはHPを持つ
* 敵との接触でダメージを受ける
* 敵弾に当たるとダメージを受ける
* ダメージ量に乱数は使用しない
* ダメージ後は短時間の無敵時間を設ける
* 無敵時間中は一定時間被弾しない
* 被弾時は被弾モーションを取り、短時間操作不能になる
* HPが0になるとゲームオーバー

## 8. 敵仕様

### 8.1 基本敵

* 敵はHPを持つ
* プレイヤーの通常弾、チャージショットでダメージを受ける
* HPが0になると消滅する
* プレイヤー接触時にプレイヤーへダメージを与える
* 敵弾を撃つ敵も存在する
* ダメージ量に乱数は使用しない
* 通常敵は一定確率で回復アイテムをドロップする

### 8.2 敵タイプ

初期実装で想定する敵タイプは以下。

* 地上固定敵
* 地上を小さくジャンプしながら移動する敵
* 空中を移動する敵
* スクロールで即消えず、倒すまで画面内に残って動き回る敵
* 弾を撃つ敵

### 8.3 敵のスクロール処理

通常の敵は背景と同様に右から左へ流れる。

一部の敵は画面内に入った後、スクロールに流されて消えるのではなく、倒されるまで画面内で動き続ける。

## 9. 成長・強化システム

### 9.1 経験値

* 敵を倒すことで経験値リソースを獲得する
* 表現方法は未定
* ネジ
* エネルギー
* パーツ

などの世界観に合わせた名称を使用する可能性がある

* 通常敵よりボスの方が大量の経験値を持つ
* 経験値量に乱数は使用しない

### 9.2 レベルアップ

* 一定量の経験値を取得するとレベルアップする
* レベルアップ時、任意のステータスを1つ選んでレベルを上げる
* 各ステータスは個別にレベルを持つ
* ステータス値は、各ステータスごとの計算式によってレベルから算出する
* 各ステータスの計算式や係数は、実際のプレイ感を見ながら調整できるようにパラメータ化する
* 分かりやすさを優先し、ステータス表示や計算結果には基本的に小数点を使わない
* 強化内容は、ゲームバランスを大きく破壊しない範囲に抑える
* プレイヤースキルとアクション性を主軸にする

### 9.3 ステータス

レベルアップで上昇できるステータスは以下。

| ステータス    | 内容                         |
| -------- | -------------------------- |
| ダメージ量    | 通常弾とチャージショットのダメージを共通で上昇させる |
| チャージ速度   | チャージ完了までの時間を短縮する           |
| 通常弾の連射速度 | 通常弾の発射間隔を短縮する              |
| 最大HP     | プレイヤーの最大HPを上昇させる           |
| ドロップ確率   | 回復アイテムのドロップ確率を上昇させる        |
| 移動速度     | 通常時とダッシュ時のスクロール速度を共通で上昇させる |
| ジャンプ速度   | ジャンプ上昇時の初速を上昇させる           |
| 最大ジャンプ高度 | 到達可能なジャンプ高度を上昇させる          |
| ダッシュ距離   | ダッシュ継続距離を上昇させる             |

### 9.4 ジャンプ・ダッシュ系ステータス

* ジャンプ速度はジャンプ開始時の上昇速度へ影響する
* 最大ジャンプ高度は到達可能な高さへ影響する
* ジャンプ速度と最大ジャンプ高度は別ステータスとして扱う
* ダッシュ距離はダッシュ継続時間へ影響する
* 地上ダッシュでは体感差が小さい場合があるが、空中ダッシュ取得後の操作感へ大きく影響する
* 移動速度ステータスは通常時のスクロール速度に影響する
* ダッシュ速度は通常速度に係数をかけて算出する
* ダッシュ速度の係数は調整可能なパラメータとして管理する

### 9.5 ダメージ量ステータス

* 表示上は通常弾ダメージ、チャージショットダメージをそれぞれ記載する
* ステータス上昇自体は共通で、両方のダメージが上昇する
* 通常弾の単発ダメージがチャージショットの単発ダメージを上回らないようにする
* チャージショットのダメージは通常弾ダメージに係数をかけて算出する
* チャージショットのダメージ係数は調整可能なパラメータとして管理する
* ダメージ量に乱数は使用しない

### 9.6 回復アイテム

* 通常敵を倒したとき、一定確率で回復アイテムをドロップする
* ドロップ確率はプレイヤー強化によって上昇できる
* 回復量は固定値を基本とする
* 回復アイテム取得時は即時回復する
* 回復アイテムはストックしない
* ドロップした回復アイテムは接触不要で必ず回収される
* 回復アイテムの取得に操作や移動を要求しない

### 9.7 ボス戦

* 一定距離ごとにボス敵が出現する
* ボス戦中は通常戦闘とは異なる演出へ切り替える
* ボスは通常敵より高HP、高火力とする
* ボスを倒すことで特別な報酬を獲得できる

### 9.8 ボス報酬

* ボス撃破後、3択の報酬を提示する
* 3択のうち1つは、そのボス固有の特殊武器を固定で提示する
* 残り2つはランダムな強化候補を提示する
* プレイヤーは3択のうち1つのみ取得できる
* 特殊武器を取るか、汎用強化を優先するかの選択を発生させる
* 通常レベルアップでは得られない強力な効果を持つ
* 一部は極端なパラメータ上昇を持つ
* ランごとに異なるビルド体験を作る
* ボス報酬は基本的に重複取得可能とする

### 9.9 特殊機能系強化

特殊機能系強化として、以下のような効果を想定する。

* ジャンプ回数を1回増やす
* 通常弾の上限を2増やす
* 通常弾の連射速度上昇
* 空中ダッシュを可能にする
* 上下方向への空中ダッシュを可能にする
* 弾サイズを大きくする
* 被弾時無敵時間の延長

これらの強化は基本的に重複取得可能とする。

* ジャンプ回数増加は無制限に重複取得可能とする
* 空中で使用できる追加アクション回数はジャンプ回数に依存する
* 例: ジャンプ回数が3回の場合、地上ジャンプ後の空中では追加で2回分の行動ができる
* 追加行動は、空中ジャンプと空中ダッシュで共有する
* 例: 空中で2回ジャンプ、2回エアダッシュ、1回ジャンプ + 1回エアダッシュのいずれも可能
* 空中で使用した追加行動回数は着地時に回復する

### 9.10 見た目変化

特殊機能系強化を取得した際、プレイヤーキャラクターの見た目も変化させる。

部位は以下の4種類を想定する。

* 頭

* 胴体

* 手

* 足

* 新しく取得した見た目で上書き表示する

* 見た目が上書きされても、過去に取得した効果は保持する

* 見た目変化によって強化取得を視覚的に分かりやすくする

* 見た目変化はレトロアクションゲームらしいパワーアップ演出として扱う

### 9.11 特殊武器

* 特殊武器はボスごとに固有のものを持つ
* 特殊武器は通常武器とは異なる攻撃特性を持つ
* 詳細仕様は未定

### 9.12 ボス報酬例

* チャージショット貫通
* 通常弾サイズ拡大
* ダッシュ中被ダメージ軽減
* HP大幅増加
* チャージ時間大幅短縮
* 通常弾同時発射数増加
* 敵撃破時HP回復強化
* 一定時間ごとのシールド

ボス報酬は強力ではあるが、ゲーム全体を完全に破壊するほどの極端なローグライト化は行わない。

## 10. ステージ仕様

### 10.1 基本構成

* 横スクロール型の進行
* 初期仕様では複雑な地形差は作らない
* 敵、敵弾、障害物を中心に難易度を作る
* ゴール制ではなく、到達距離を主な成果とする
* スコアは到達距離のみを基本とする

### 10.2 ステージデータ

ステージはJSONまたはTypeScriptのデータとして管理する。

初期はTypeScriptのオブジェクトで簡易実装してもよい。

```ts
type StageData = {
  id: string;
  name: string;
  baseScrollSpeed: number;
  enemySpawns: EnemySpawnData[];
};

type EnemySpawnData = {
  distance: number;
  enemyType: string;
  y: number;
};
```

## 12. アセット方針

### 12.1 画像

* 最終的にはオリジナル素材を使用する
* スプライト、背景などの方向性は未定
* 既存ゲーム素材は使用しない
* 開発初期は丸、四角、単色図形などの仮素材で実装する
* AI生成素材を使う場合も、既存作品に酷似しすぎないようにする

### 12.2 音声

* 最終的にはオリジナルSE/BGMを使用する
* 既存ゲーム音源は使用しない
* 初期実装では音声なしでもよい
* すぐに実装できる場合は、AudioWorkletまたはWeb Audio APIで簡易SEを生成する
* フリー素材を使う場合はライセンス確認必須

## 13. パラメータ管理方針

* ゲームバランス調整用の数値は一箇所へ集約して管理する
* 実装コードへ直接マジックナンバーを書かない
* プレイ感を見ながら調整しやすい構造を優先する
* ステータス計算式や係数もパラメータとして外部化する
* 初期実装では TypeScript の定数ファイルで管理する

想定例:

```ts
export const GAME_BALANCE = {
  player: {
    baseHp: 10,
    baseMoveSpeed: 300,
    dashSpeedMultiplier: 1.8,
    dashDuration: 180,
    damageInvincibleTime: 60,
    chargeFrame: 90,
  },

  weapon: {
    bulletLimit: 3,
    baseBulletDamage: 1,
    chargeShotMultiplier: 5,
  },

  levelScaling: {
    damagePerLevel: 1,
    hpPerLevel: 2,
  },
};
```

## 15. デプロイ

CloudflareへWranglerでデプロイする。

想定コマンド:

```bash
pnpm build
pnpm wrangler deploy
```

静的配信で足りる場合は Cloudflare Pages を使う。Worker側の処理が必要になった場合は Cloudflare Workers に寄せる。
